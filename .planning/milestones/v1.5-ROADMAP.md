# Milestone v1.5: Chat-First Copilot

**Status:** SHIPPED
**Archived:** 2026-04-18
**Phase range:** 10-11
**Total plans:** 8

## Overview

This milestone unified Seeku's shipped conversational surfaces into one chat-first product. `/chat` became the primary operator surface, the right rail became a narrated session workboard derived from authoritative runtime state, and a bounded foreground mission runner was added for large-scope candidate search without weakening compare gating or recommendation honesty.

## Included Phase

### Phase 10: Chat-First Copilot

**Goal:** Turn `/chat` into the primary copilot surface with a read-only narrated workboard that reuses the existing runtime protocol and absorbs deal-flow/session artifacts without weakening runtime authority.

**Depends on:** Phase 08 `cli-agent-panel`, Phase 09 `daily-deal-flow`

**Plans:**
- [x] `10-01-PLAN.md` — Chat-first shell and authoritative session binding
- [x] `10-02-PLAN.md` — Narrated workboard state mapping and read-only information architecture
- [x] `10-03-PLAN.md` — Focus views, transitional route reuse, and UI integration
- [x] `10-04-PLAN.md` — Fallback handling, regression coverage, and milestone acceptance

### Phase 11: Bounded Mission Runner

**Goal:** Add a foreground, bounded long-task runner for large-scope candidate search inside the chat-first copilot, with explicit stop rules and in-flight course correction.

**Depends on:** Phase 10 `chat-first-copilot`

**Plans:**
- [x] `11-01-PLAN.md` — Mission object, session attachment, and bounded state machine foundation
- [x] `11-02-PLAN.md` — Mission loop policy, stop rules, and course-correction protocol
- [x] `11-03-PLAN.md` — Mission UI framing inside chat and workboard
- [x] `11-04-PLAN.md` — Regression coverage, stop-quality verification, and milestone acceptance

## Delivered Outcome

- Promoted `/chat` to the default product entry and operator surface.
- Added a narrated, read-only right rail with:
  - `Now`
  - `Why`
  - `Movement`
  - `Focus`
- Folded shortlist, compare posture, recommendation posture, and proactive deal-flow output into one session-centric surface.
- Redirected legacy `/agent-panel/[sessionId]` usage into the chat-first shell.
- Added a bounded foreground mission runner for large-scope candidate search that:
  - runs for multiple rounds in one session
  - accepts natural-language `tighten`, `retarget`, and stop/pause corrections
  - stops explicitly with:
    - `enough_shortlist`
    - `enough_compare`
    - `low_marginal_gain`
    - `needs_user_clarification`
- Hardened mission stop posture so reportable output is distinct from recommendable output.

## Key Decisions

- Keep the CLI/runtime layer as the single source of truth.
- Make chat the only primary steering thread; keep the right rail read-only.
- Treat deal flow as a session artifact, not a second runtime mode.
- Keep the mission runner foreground-bound and single-mission-per-session.
- Default mission stop posture to shortlist/compare-first rather than top1-first.

## Deferred / Watch Items

- mission-quality acceptance still relies on deterministic replay-style scenarios rather than live search replays
- the mission runner is only accepted for the first mission type:
  - large-scope candidate search
- no background continuation, multi-mission inbox, CRM workflow, or durable task history is part of v1.5
- worker-side `Q4/Q6/Q8` posture remains a carry-forward guardrail and was not rerun in this web-scoped milestone close
