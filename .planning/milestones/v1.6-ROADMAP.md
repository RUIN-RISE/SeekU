# Milestone v1.6: Mission Replay Hardening

**Status:** SHIPPED
**Archived:** 2026-04-18
**Phase range:** 12
**Total plans:** 4

## Overview

This milestone hardened the shipped bounded mission runner with a replay-backed quality gate. The goal was not to add new mission categories or broaden automation scope, but to move mission-quality checks from scattered deterministic scenarios toward reusable replay fixtures, explicit failure buckets, and bounded replay-driven semantic fixes.

## Included Phase

### Phase 12: Mission Replay Hardening

**Goal:** Build a replay-backed mission-quality gate for the shipped bounded mission runner, classify stop-quality failures cleanly, and accept only bounded semantic fixes before broadening the mission product surface.

**Depends on:** Phase 10 `chat-first-copilot`, Phase 11 `bounded-mission-runner`

**Plans:**
- [x] `12-01-PLAN.md` — Replay case inventory, fixture format, and harness foundation

- [x] `12-03-PLAN.md` — Replay-driven semantic fixes and stop-quality alignment
- [x] `12-04-PLAN.md` — Replay-backed acceptance, residual-risk report, and milestone close

## Delivered Outcome

- Added reusable replay fixtures for the shipped bounded search mission type.
- Moved key mission stop scenarios into a shared replay-case structure.
- Added explicit replay evidence capture for:
  - stop reason
  - mission phase
  - summary copy
  - uncertainty copy
  - shortlist size
  - compare size
- Added explicit failure buckets:
  - `false_stop`
  - `late_stop`
  - `wrong_stage_report`
  - `ui_semantic_mismatch`
- Used replay evidence to drive a bounded semantic fix:
  - clarification stops now prioritize `Goal summary` in the right rail instead of drifting into shortlist-first focus

## Key Decisions

- Use replay fixtures and explicit mismatch buckets before broadening mission scope.
- Keep replay hardening limited to the shipped first mission type:
  - large-scope candidate search
- Prefer bounded semantic fixes over speculative stop-policy branching.
- Treat replay hardening as a quality gate, not as an excuse to add background orchestration or new workflow surfaces.

## Deferred / Watch Items

- replay cases are still deterministic fixtures rather than harvested live-search replays
- replay evidence is still test-local rather than emitted as a standalone artifact file
- non-replay correction scenarios are still valuable but not yet folded fully into the shared replay inventory
- no new mission types, background continuation, multi-mission management, or CRM workflow were added in this milestone
