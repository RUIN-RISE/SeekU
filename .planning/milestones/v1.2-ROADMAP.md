# Milestone v1.2: Agentic Search CLI

**Status:** SHIPPED
**Archived:** 2026-04-16
**Phase range:** 7
**Total plans:** 4

## Overview

This milestone added a CLI-first agent layer above Seeku's existing evidence-driven search core. The delivery goal was to let the CLI autonomously clarify, search, narrow, compare 2-3 candidates, and only recommend when evidence and confidence gates supported the decision.

## Included Phase

### Phase 7: CLI Search Agent Orchestration

**Goal:** Evolve Seeku CLI into a decision-oriented search agent that can autonomously clarify, search, compare 2-3 candidates, and recommend with evidence and confidence gates.

**Plans:**
- [x] `07-01-PLAN.md` — Agent tools and session state foundation
- [x] `07-02-PLAN.md` — Structured compare engine and confidence gates
- [x] `07-03-PLAN.md` — Free-form CLI agent policy and interaction loop
- [x] `07-04-PLAN.md` — Agent eval harness and acceptance validation

## Delivered Outcome

- Added explicit agent-callable tool contracts above the existing search core.
- Centralized session state for user goal, normalized conditions, shortlist, compare set, confidence state, and open uncertainties.
- Added fixed compare dimensions and explicit outcomes:
  - `clear-recommendation`
  - `conditional-recommendation`
  - `no-recommendation`
- Added a bounded policy layer that chooses among `clarify`, `search`, `narrow`, `compare`, and `decide`.
- Added `agent-eval` for repeatable acceptance and regression validation.

## Key Decisions

- Keep `planner / retriever / reranker` as the evidence engine and add orchestration above it instead of replacing it.
- Treat compare as the mandatory pre-recommendation gate.
- Prefer refusal or conditionality over unsupported certainty.
- Reuse the saved controlled-open WS4 snapshot pack as the regression baseline instead of creating a new ranking benchmark.
