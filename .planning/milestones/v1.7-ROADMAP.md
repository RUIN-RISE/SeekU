# Milestone v1.7: Runtime-Backed Chat Agent Integration

**Status:** SHIPPED
**Archived:** 2026-04-18
**Phase range:** 13
**Total plans:** 4

## Overview

This milestone replaced front-end simulated mission execution in `/chat` with a bounded runtime/API-backed authority path. The goal was not to broaden Seeku into a general public agent API or a second browser-owned runtime, but to make the shipped chat-first copilot run against real runtime-backed sessions while preserving right-rail framing, stop semantics, and recommendation honesty.

## Included Phase

### Phase 13: Runtime-Backed Chat Agent Integration

**Goal:** Route `/chat` through a real runtime/API-backed agent execution path for the current bounded mission type while preserving the existing chat-first UX and mission semantics.

**Depends on:** Phase 10 `chat-first-copilot`, Phase 11 `bounded-mission-runner`, Phase 12 `mission-replay-hardening`

## Delivered Outcome

- `/chat?sessionId=...` now binds to an attached runtime-backed session instead of pretending local hook-owned execution is authoritative.
- Added `POST /chat-missions` plus worker bootstrap wiring so the integrated chat path can start a real runtime-backed mission from the initial prompt.
- Runtime snapshots and events are projected into the existing chat-first mission/workboard shape, so the shipped UI contract stays intact while authority moves to the runtime.
- Attached chat now supports bounded runtime-backed correction through the existing intervention surface and derives correction notices from runtime events instead of local optimistic bookkeeping.
- Missing, disconnected, and error states stay explicit and do not silently fall back to local simulated mission execution.
- The milestone closed with a targeted acceptance artifact and rollout guardrails for the first bounded runtime-backed chat scope.

## Key Decisions

- Keep the runtime as the single source of truth for integrated `/chat` sessions.
- Preserve the shipped chat-first shell and right-rail contract instead of adding a new operator console.
- Keep attached chat correction bounded to the existing runtime intervention surface in this milestone.
- Keep rollout internal-first and do not broaden this milestone into a general public runtime API.

## Deferred / Watch Items

- attached chat correction is still bounded to feedback-tag-style runtime interventions rather than full freeform correction parsing
- direct stop/pause requests from attached chat are not yet translated into a runtime command path
- the accepted scope is still the first bounded search mission type rather than broader agent workflow orchestration
- broader worker-side search-eval tracks remain separate from this authority-handoff milestone
