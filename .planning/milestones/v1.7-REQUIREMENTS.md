# Requirements Archive — Seeku v1.7

Archived: 2026-04-18
Milestone: `v1.7 Runtime-Backed Chat Agent Integration`
Status: Complete

## Archived Requirements

### Runtime Authority

- [x] `RUNTIME-01` — `/chat` starts a real runtime-backed session or mission through an internal API instead of relying only on local hook-owned execution.
- [x] `RUNTIME-02` — Mission progress, stop reason, and final summary shown in chat are derived from runtime/API authority rather than front-end fabricated state.
- [x] `RUNTIME-03` — The right rail consumes runtime-backed session state while preserving the shipped `Now / Why / Movement / Focus` contract.
- [x] `RUNTIME-04` — The integration preserves the rule that the runtime remains the single source of truth for shortlist, compare posture, recommendation posture, and open uncertainties.

### Interaction Loop

- [x] `LOOP-01` — The first integrated loop supports: start mission, stream progress, send correction, and stop with summary.
- [x] `LOOP-02` — Natural-language corrections from `/chat` are routed to the runtime/API layer and stay inside the same session/mission.
- [x] `LOOP-03` — The integrated path preserves existing stop categories and user-facing stop semantics for the shipped bounded search mission type.

### Reliability And Rollout

- [x] `ROLL-01` — The product degrades safely when the runtime/API path is unavailable, interrupted, or missing a live session.
- [x] `ROLL-02` — The integration remains internal-first; this milestone does not turn the runtime into a general external agent API product.
- [x] `ROLL-03` — Existing targeted web regressions remain green after the runtime-backed path is added.

## Outcome Notes

- `RUNTIME-01` to `RUNTIME-04`: validated through the `/chat-missions` start path, attached runtime projection, and the runtime-authoritative chat/workboard integration
- `LOOP-01` to `LOOP-03`: validated through the bounded start/stream/correction/summary loop accepted in Phase 13
- `ROLL-01` to `ROLL-03`: validated through explicit degraded-state handling, bounded rollout guardrails, and the final targeted regression suite
