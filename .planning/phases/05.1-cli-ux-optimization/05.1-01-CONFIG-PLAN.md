# 05.1-01: Config 统一配置 + Zod 验证

## Goal
Centralize all CLI constants and enable environment-based overrides with strict schema validation.

## Proposed Changes
### [NEW] [config.ts](file:///Users/rosscai/seeku/apps/worker/src/cli/config.ts)
- Define `CLI_CONFIG` using `zod`.
- Schema:
  - `llm`: `timeoutMs` (8000), `maxRetries` (2), `parallelLimit` (3)
  - `scoring`: `weights` (techMatch, projectDepth, etc.), `experienceBonusCap` (30)
  - `cache`: `ttlDays` (7)
  - `ui`: `defaultLimit` (10), `spinnerEnabled` (true), `inputTimeoutMs` (120000)
- Support `process.env` mapping (e.g., `SEEKU_LLM_TIMEOUT`).
- Immediate `parse()` to ensure CI/startup failure on invalid config.

### [MODIFY] [scorer.ts](file:///Users/rosscai/seeku/apps/worker/src/cli/scorer.ts)
- Replace local `SCORING_WEIGHTS` and `LLM_TIMEOUT_MS` with imports from `config.ts`.

### [MODIFY] [workflow.ts](file:///Users/rosscai/seeku/apps/worker/src/cli/workflow.ts)
- Replace local `INPUT_TIMEOUT_MS` and `limit: 50` with `CLI_CONFIG`.

## Verification
- `pnpm build`
- Run CLI with `SEEKU_LLM_TIMEOUT=abc` and verify it fails with a clear Zod error.
