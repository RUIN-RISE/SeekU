<decisions>
## Implementation Decisions

### 加载状态（P0 - 必须完成）
- 使用 ora spinner 显示 LLM 调用进度
- spinner 仅用于非交互等待阶段（LLM调用、缓存计算）
- 在 `tui.selectCandidate()` 调用前必须 `spinner.stop()`
- ora 使用 stderr，enquirer 使用 stdout raw mode，需确保交替使用时 stop() 完成
- spinner 文案随阶段变化："缓存检查..." → "计算评分..." → "生成洞察..."

### 并行预加载（P0 - 必须完成）
- 使用懒预加载 + 即时优先策略
- 后台预加载不阻塞用户交互循环
- `promisePool` 必须接收工厂函数 `(() => Promise<T>)[]`，不是 `Promise<T>[]`
- 并发限制默认 3，可配置
- 用户退出时静默取消预加载

### 配置管理（P1 - 建议完成）
- 创建 `cli/config.ts` 统一管理所有配置
- 使用 Zod Schema 验证配置（防止 NaN）
- 支持环境变量覆盖（SEEKU_LLM_TIMEOUT, SEEKU_LLM_RETRIES 等）
- 启动时 fast-fail 验证

### 重试机制（P1 - 建议完成）
- 创建 `cli/retry.ts` 提供重试工具
- 区分可重试错误（429/503/AbortError/ETIMEDOUT）和不可重试错误（401/400）
- 使用指数退避策略
- 重试日志显示进度

### Claude's Discretion
- spinner 类型选择（dots/line/moon）
- 具体 spinner 文案设计
- 预加载触发时机细节
- 错误日志格式

</decisions>

### promisePool 正确实现
```typescript
async function promisePool<T>(
  tasks: (() => Promise<T>)[],  // 工厂函数！
  concurrency: number
): Promise<T[]> {
  const results: T[] = [];
  const executing: Promise<void>[] = [];

  for (const task of tasks) {
    const p = task().then(result => {
      results.push(result);
      executing.splice(executing.indexOf(p), 1);
    });
    executing.push(p);

    if (executing.length >= concurrency) {
      await Promise.race(executing);
    }
  }

  await Promise.all(executing);
  return results;
}
```

### isRetryable 错误区分
```typescript
function isRetryable(error: Error): boolean {
  if (error.name === 'AbortError') return true;
  if (error.message.includes('429')) return true;
  if (error.message.includes('503')) return true;
  if (error.message.includes('401')) return false;
  if (error.message.includes('400')) return false;
  return true;
}
```
