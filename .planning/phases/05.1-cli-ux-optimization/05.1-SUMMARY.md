## Objective

Transform Seeku CLI from "single-shot search tool" into "conversational search assistant" with a proper session loop.

---

## What Was Built

### 1. Session Loop Architecture

**Before:**
```
ask skills → refine → search → select → detail → exit
```

**After:**
```
clarify → search → shortlist → detail/compare/refine → search (loop)
```

### 2. Entry Point Improvements

| Command | Behavior |
|---------|----------|
| `seeku` | Enters interactive session directly |
| `seeku "query"` | Carries initial requirement into session |
| `seeku search` | Non-JSON mode enters session |
| `seeku --help` | Shows assistant-style help |
| `seeku --version` | Shows v1.1.0 |

### 3. Pre-Search Clarification

User input triggers **understanding echo**:
- Current interpretation: role, skills, location, experience
- Missing fields highlighted
- User actions: `[1] search [2] add [3] relax [4] restart [q] quit`

### 4. Command-Based Shortlist

Replaced `Select` UI with command interface:
```
Top 5 / 20 | 排序: 综合分
========================================================================
1. Jason  75.5
   杭州 · Senior AI Engineer
   为什么匹配：熟悉 CUDA 和 vLLM，符合推理优化需求
========================================================================
动作：v 2 查看详情 | c 1 3 对比 | sort tech | r 重新收敛 | m 更多 | q 退出
```

### 5. Detail Page Actions

Fixed post-detail dead-end:
```
动作：back 返回结果页 | why 看评分依据 | refine 继续收敛 | q 退出
```

### 6. Infrastructure Improvements

| Feature | Implementation |
|---------|---------------|
| Loading states | ora spinner with phase-aware text |
| Parallel preload | promisePool factory pattern |
| Retry mechanism | withRetry + isRetryable classification |
| Query-scoped cache | SHA1 hash of conditions → separate cache entries |
| Config management | Zod schema validation, env var override |

## Post-Completion Enhancements (2026-03-30)

### New Features Added

| Feature | Description |
|---------|-------------|
| **matchReason 强化** | 基于评分维度生成，连接 `scoreByRules` 结果 |
| **Compare Pool** | `add N`, `pool`, `clear` 命令，持久化候选人对比 |
| **Session Memory** | `history`, `undo`, `show` 命令，搜索历史管理 |
| **dotenv 路径修复** | ESM 兼容方式加载根目录 `.env` |

### Command Reference

| Command | Description |
|---------|-------------|
| `add N` | 把第 N 位候选人加入对比池 |
| `pool` | 查看当前对比池 |
| `clear` | 清空对比池 |
| `c` | 对比池内候选人（需 pool ≥ 2） |
| `c 1 3` | 对比第 1 和第 3 位 |
| `history` | 查看搜索历史 |
| `undo` | 回到上一轮搜索条件 |
| `show` | 显示当前筛选条件 |

### Bug Fixes

| Issue | Fix |
|-------|-----|
| `c` 无法对比池内候选人 | 解析器返回 `compare` 类型，workflow 检测使用 pool |
| undo 不精确恢复 | 新增 `restore` 类型，直接恢复 conditions，不走 LLM |
| compare pool 条件一致性 | 对 pool 候选人清除旧 profile，强制重新生成 |
| dotenv 找不到 .env | 使用 ESM 兼容路径解析 |
