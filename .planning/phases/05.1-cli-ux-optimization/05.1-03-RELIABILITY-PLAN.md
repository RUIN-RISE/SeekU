# 05.1-03: Reliability 生产环境健壮性 (retry.ts)

## Goal
Replace immediate LLM failure with a robust retry mechanism using exponential backoff and error classification.

## Proposed Changes
### [NEW] [retry.ts](file:///Users/rosscai/seeku/apps/worker/src/cli/retry.ts)
- `withRetry<T>(fn: () => Promise<T>, options: RetryOptions): Promise<T>`
- `isRetryable(error: unknown): boolean`
  - Retry: 429, 503, AbortError, ETIMEDOUT.
  - Fail: 400, 401.

### [MODIFY] [scorer.ts](file:///Users/rosscai/seeku/apps/worker/src/cli/scorer.ts)
- Wrap `scoreByLLM` in `withRetry`.
- Update `scoreByLLM` to use `CLI_CONFIG.llm.maxRetries`.

### [MODIFY] [profile-generator.ts](file:///Users/rosscai/seeku/apps/worker/src/cli/profile-generator.ts)
- Wrap `generate` in `withRetry`.
