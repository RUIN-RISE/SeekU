# Phase 05.1 CLI 验收修复总结

**时间:** 2026-03-30  
**阶段:** Phase 05.1: CLI UX Optimization  
**触发来源:** Claude review findings remediation

## 本轮修复目标

针对验收阶段暴露的 5 个关键 CLI 问题进行直接修复：

1. 交互模式启动失败时只输出 JSON 错误，缺乏 CLI 友好提示
2. 候选人画像缓存仅按 `personId` 命中，跨查询条件复用错误结果
3. 画像摘要生成缺少超时控制，详情页与后台预热会卡住
4. 跳过经验要求后把“不限”拼进真实搜索 query
5. 追问超时后未真正关闭 enquirer prompt，存在 stdin 残留风险

## 已完成修复

### 1. 启动失败改为 CLI 友好提示

- 文件: `apps/worker/src/cli/index.ts`
- 修复内容:
  - 将 LLM provider 和数据库连接初始化移动到 `try` 内部
  - 对缺失 `SILICONFLOW_API_KEY` / `OPENAI_API_KEY` 与 `DATABASE_URL` 给出明确提示
  - 交互模式失败时不再回退到顶层 JSON 错误输出

### 2. 画像缓存改为查询条件隔离

- 文件: `apps/worker/src/cli/workflow.ts`
- 文件: `packages/db/src/profile-cache.ts`
- 修复内容:
  - 为 `SearchConditions` 生成稳定的 query-scoped cache key
  - `processingProfiles` 改为按 `personId + queryKey` 去重
  - 保持现有 `profile_cache` 表结构不变，在单行 JSONB 中存储多份按条件隔离的缓存 entry
  - 旧版单画像缓存视为 legacy，不再跨查询复用

### 3. 画像摘要生成补齐超时和取消

- 文件: `apps/worker/src/cli/profile-generator.ts`
- 修复内容:
  - 为 profile summary 生成链路加入独立 `AbortController`
  - 统一接入 `CLI_CONFIG.llm.timeoutMs`
  - 超时时返回降级 summary，而不是无限挂起

### 4. 跳过经验要求不再污染检索 query

- 文件: `apps/worker/src/cli/chat.ts`
- 文件: `apps/worker/src/cli/workflow.ts`
- 修复内容:
  - 用户跳过经验追问时不再把 `experience` 写成 `"不限"`
  - `buildEffectiveQuery()` 额外过滤 `"不限" / "skip" / "none"` 之类的哨兵值，避免误入真实检索

### 5. 追问超时后主动清理 prompt

- 文件: `apps/worker/src/cli/chat.ts`
- 修复内容:
  - 超时分支会主动调用 `promptBuffer.cancel()`
  - 等待 prompt promise 清理完成后再继续后续交互
  - 避免旧 prompt 继续占用 stdin
