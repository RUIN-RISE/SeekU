# 05.1-04: Performance 并行预加载 (promisePool)

## Goal
Minimize waiting time when a user selects a candidate by pre-calculating profiles in the background while the user is browsing the candidate list.

## Proposed Changes
### [MODIFY] [workflow.ts](file:///Users/rosscai/seeku/apps/worker/src/cli/workflow.ts)
- Implement `promisePool<T>(tasks: (() => Promise<T>)[], concurrency: number): Promise<T[]>`
- In `execute()`:
  - After `performSearch`, start `preloadProfiles(candidates, conditions)` as a background task.
- In `interactionLoop()`:
  - Background task runs concurrently with `tui.selectCandidate()`.
- In `showCandidateDetail()`:
  - If `cacheRepo.getProfile(id)` exists (likely preloaded), it skips LLM calls.

## Verification
- View candidate details; verify "Cache hit!" is logged immediately.
- Trace logs to confirm "Preloading..." starts while user is selecting.
- Verify `parallelLimit` for LLM calls is respected during preloading.
