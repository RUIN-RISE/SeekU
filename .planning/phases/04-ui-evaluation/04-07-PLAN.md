  <how-to-verify>
**Prerequisites:**
1. Database must have test data (persons with evidence items)
2. API server must be running: `pnpm --filter @seeku/api start`
3. Web dev server must be running: `pnpm --filter @seeku/web dev`

**Verification Steps:**

1. **API Endpoints Test:**
   ```bash
   # Test profiles endpoint (replace with actual personId from database)
   curl http://localhost:3000/profiles/<personId>

   # Test admin sync status
   curl http://localhost:3000/admin/sync-status

   # Test admin run-eval placeholder
   curl -X POST http://localhost:3000/admin/run-eval
   ```

2. **CLI Search Test:**
   ```bash
   # Test search command
   pnpm --filter @seeku/worker search "AI工程师 Python" --json

   # Test show command (replace with actual personId)
   pnpm --filter @seeku/worker show <personId> --json
   ```

3. **Web Frontend Test:**
   - Open http://localhost:3001 (or Next.js default port) in browser
   - Verify dark header with "Seeku" logo
   - Verify Chinese title "发现AI人才" displayed
   - Enter search query in SearchBar (wait 300ms debounce)
   - Verify results appear in grid layout
   - Click a candidate card to open modal
   - Verify modal shows candidate name, headline
   - Verify evidence tabs (Projects, Repositories, Socials, Job Signals)
   - Navigate to /admin page
   - Verify sync status card shows data
   - Verify eval metrics card displays (may show "No eval results yet")

4. **Visual Check:**
   - Header: Dark background (#0a1628), light text
   - Cards: White background, shadow, hover lift
   - Score badge: Dark background, rounded pill
   - Tags: Blue accent background
   - Modal: Backdrop blur, scale-in animation

5. **Eval Package Test:**
   ```bash
   # Verify eval package builds
   pnpm --filter @seeku/eval build

   # Verify datasets exist
   ls packages/eval/datasets/queries.json
   ls packages/eval/datasets/golden-set.json

   # Count queries
   cat packages/eval/datasets/queries.json | python3 -c "import json,sys; print(len(json.load(sys.stdin)))"
   ```
