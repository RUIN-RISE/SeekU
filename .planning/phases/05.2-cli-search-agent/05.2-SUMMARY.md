## Objective

Transform Seeku CLI from "conversational search assistant" into a true "CLI Search Agent" that helps users complete the full hiring discovery loop:

`Express goal → Converge needs → Shortlist → Explain → Guide next action`

The core goal: **Let users perceive Bonjour's value.**

---

## What Was Built

### 1. Source Visibility

Every shortlist item now shows where the data comes from:

| Badge | Meaning |
|-------|---------|
| `[Bonjour]` | Data from Bonjour.bio |
| `[GitHub]` | Data from GitHub |
| `来源未知` | No source metadata |

Implementation:
- Derive `sources` from `searchDocument.facetSource`
- Fallback: if `bonjourUrl` exists, mark as `["Bonjour"]`
- Never show "Unknown" when Bonjour link is present

### 2. Freshness Indicators

Every candidate shows data freshness:

| Display | Meaning |
|---------|---------|
| `新鲜 0天` | Evidence/sync within 7 days |
| `30天前` | Evidence/sync 30 days ago |
| `3个月前` | Older evidence |

Implementation:
- `latestEvidenceAt`: most recent evidence timestamp
- `lastSyncedAt`: person record update time
- Human-friendly relative date formatting

### 3. Bonjour Link Display

Every shortlist item shows Bonjour profile link:
```
🔗 https://bonjour.bio/jq8ny1
```

Implementation:
- Query `personIdentities` → `sourceProfiles`
- Extract `canonicalUrl` where `source === "bonjour"`

### 4. Open Bonjour Action

New command to open Bonjour profile in browser:

| Context | Command |
|---------|---------|
| Shortlist | `o 2` - open candidate #2's Bonjour page |
| Detail | `o` or `open` - open current candidate's Bonjour page |

Implementation:
- Platform-aware browser launch (`open` / `start` / `xdg-open`)
- Detached spawn, doesn't block CLI
- Graceful fallback if Bonjour link missing

### 5. Detail Page Enhancement

Detail view now includes complete source information:

```
数据来源： [Bonjour]
🔗 Bonjour: https://bonjour.bio/xxx
Last Synced: 7天前 | Latest Evidence: 今天
Evidence Sources: Bonjour, GitHub
```

## Key Fix (Post-Initial Implementation)

**Issue:** Source badge showed `来源未知` even when Bonjour link was present.

**Root Cause:** `sources` only derived from sparse `facetSource`, not from identity.

**Fix:** If `bonjourUrl` exists but `facetSource` is empty, backfill `sources` with `["Bonjour"]`.

**Locations:**
- `workflow.ts:698` - main search path
- `workflow.ts:845` - fallback search path
