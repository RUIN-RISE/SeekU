## 2. sameComponentBoost 最终处理决定

### 2.1 决定：禁用 (boost = 0%)

### 2.2 原因

1. **数据事实**: 99.8% 的图节点在一个 giant component 里
2. **无区分度**: `sameComponentAsAnchor` 对几乎所有 candidate 都是 true
3. **无意义 boost**: 1% 的 boost 对所有 candidate 都适用，等于没有 boost
4. **误导性 match reason**: "graph: same component" 会让用户误以为有特殊关联

### 2.3 实现方式

```typescript
// scoring-config.ts
graphSameComponentBoost: envNumber("SEEKU_GRAPH_SAME_COMPONENT_BOOST", 0)  // 默认 0

// reranker.ts - match reason 只在 boost > 0 时显示
if (graphFeatures.sameComponentAsAnchor && this.config.graphSameComponentBoost > 0) {
  reasons.push("graph: same component");
}
```

### 2.4 可逆性

如果未来图结构变化（例如多个独立社区），可通过环境变量重新启用：
```bash
SEEKU_GRAPH_SAME_COMPONENT_BOOST=0.01
```

## 3. Feature Flag / Telemetry 设计

### 3.1 Feature Flag

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `SEEKU_GRAPH_RERANK_ENABLED` | 0 (disabled) | Master switch - 设为 1 显式启用 graph rerank |
| `SEEKU_GRAPH_MUTUAL_CONNECTION_BOOST` | 0.02 | 每个 mutual connection 的 boost (2%) |
| `SEEKU_GRAPH_DIRECT_NEIGHBOR_BOOST` | 0.08 | 一度人脉 boost (8%) |
| `SEEKU_GRAPH_SAME_COMPONENT_BOOST` | 0 | 同组件 boost (已禁用) |

### 3.2 Telemetry Logging

**日志格式**: `[GraphRerank]` JSON 结构化输出

```json
// 每次搜索输出
[GraphRerank] {
  "enabled": true,
  "hasAnchor": true,
  "candidateCount": 20,
  "graphFeatureCount": 15,
  "graphFeatureRate": 0.75
}

// 有 boost 时额外输出
[GraphRerank:Boosts] {
  "boostedCount": 13,
  "top5GraphReasons": [
    {"id": "1a00b988", "score": "0.712", "graphReasons": ["graph: 3 mutual connections"]},
    ...
  ]
}
```

### 3.3 监控指标

| 指标 | 期望值 | 异常信号 |
|------|--------|----------|
| `hasAnchor` | ~50% of searches | 如果总是 false，说明 anchor context 未正确传递 |
| `graphFeatureRate` | ~75% for anchored queries | 如果过低，说明图数据覆盖不足 |
| `boostedCount` | 10-14 per query | 如果过高，可能存在 popularity bias |

## 6. 剩余风险和回滚方式

### 6.1 已知风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 25% candidate 无图数据 | 无 boost，可能被低估 | 监控 `graphFeatureRate`，考虑添加说明 |
| BFS 采样偏差 | 高度数节点可能过度代表 | 保守 boost 值，监控 popularity bias |
| 边语义限制 | "follow/follower" 非信任关系 | Match reason 使用准确语义 |
| 无时间戳 | 无法判断连接新旧 | 不做时间相关推断 |

### 6.2 回滚方式

**立即回滚**:
```bash
# 环境变量方式
export SEEKU_GRAPH_RERANK_ENABLED=0

# 或在 .env 中
SEEKU_GRAPH_RERANK_ENABLED=0
```

**调整 boost 值**:
```bash
# 降低 mutual connection boost
export SEEKU_GRAPH_MUTUAL_CONNECTION_BOOST=0.01

# 降低 direct neighbor boost
export SEEKU_GRAPH_DIRECT_NEIGHBOR_BOOST=0.04
```

### 6.3 回滚指标

如果出现以下情况，应考虑回滚：
- Non-graph queries 出现排序变化
- `graphFeatureRate` 突然下降 (图数据问题)
- 用户反馈结果质量下降
- 高度数节点垄断 top 结果
