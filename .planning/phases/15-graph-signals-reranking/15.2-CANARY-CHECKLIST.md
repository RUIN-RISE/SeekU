# Phase 15.2: Graph Signals Reranking - Canary Checklist

**Date**: 2026-05-03  
**Project**: Seeku  
**Status**: Ready for Canary

## Goal

Safely enable graph rerank in a controlled environment, verify it behaves as expected, and keep rollback one env var away.

## Deployment Contract

- Default behavior: `SEEKU_GRAPH_RERANK_ENABLED=0`
- Canary behavior: explicitly set `SEEKU_GRAPH_RERANK_ENABLED=1`
- Rollback behavior: set `SEEKU_GRAPH_RERANK_ENABLED=0`

## Preflight

1. Confirm the deployed build includes:
   - `packages/search/src/scoring-config.ts`
   - `packages/search/src/reranker.ts`
   - `apps/worker/src/cli/search-executor.ts`
2. Confirm `sameComponentBoost=0`.
3. Confirm `.env`, staging secrets, or deployment env include an explicit value for `SEEKU_GRAPH_RERANK_ENABLED`.
4. Confirm the canary environment has access to Bonjour graph tables:
   - `graph_edges`
   - `graph_node_features`
5. Confirm there is a way to inspect logs for `[GraphRerank]`.

## Canary Steps

1. Keep prod default off.
2. Enable graph rerank only in staging/canary.
3. Run a small batch of graph-sensitive searches with anchors.
4. Run a small batch of non-graph searches.
5. Compare top-5 / top-10 output against baseline.

## What To Watch

### Log lines

- `[GraphRerank:Config]`
- `[GraphRerank:Fetch]`
- `[GraphRerank:Applied]`

### Fields

- `enabled`
- `hasAnchor`
- `candidateCount`
- `graphFeatureCount`
- `graphFeatureRate`
- `boostedCount`

### Expected patterns

- Anchored queries: `enabled=true`, `hasAnchor=true`, `graphFeatureRate > 0`
- Non-graph queries: `hasAnchor=false`, `boostedCount=0`
- If `enabled=false`, graph reasons should not appear

## Pass Criteria

- Non-graph queries remain identical to baseline.
- Anchored queries show only conservative rank movement.
- `graphFeatureRate` is not unexpectedly near zero.
- `boostedCount` is non-zero only for anchored queries.
- No user-visible explanation implies trust / friendship / collaboration.

## Rollback

```bash
export SEEKU_GRAPH_RERANK_ENABLED=0
```

If needed, also reduce boost values:

```bash
export SEEKU_GRAPH_MUTUAL_CONNECTION_BOOST=0.01
export SEEKU_GRAPH_DIRECT_NEIGHBOR_BOOST=0.04
```

## Stop Conditions

- Non-graph search results shift unexpectedly.
- Anchored queries look over-boosted.
- `graphFeatureRate` collapses unexpectedly.
- Logs show graph rerank running when `enabled=false`.

## Exit Criteria

- Canary logs look stable.
- Baseline queries remain unchanged.
- Rollback was verified once.
- Team is comfortable widening exposure.
