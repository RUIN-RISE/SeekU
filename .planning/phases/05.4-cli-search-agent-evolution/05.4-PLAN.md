# Phase 05.4: CLI Search Agent Evolution - Execution Plan

**Phase:** 05.4-cli-search-agent-evolution
**Status:** 🔄 IN PROGRESS
**Completed:** P0 (Compare V2 + Export)

---

## Objective

把 Seeku CLI 从“能搜到一批人”的搜索助手，推进成一个更完整的搜寻智能体：

- 能比较候选人并帮助做判断
- 能把 shortlist 导出成可带走的结果
- 能更自然地 refine / rerank，而不是只依赖用户手写结构化条件

目标不是继续堆命令，而是补齐用户真实任务闭环：

`找人 -> 判断 -> 导出/分享 -> 打开 Bonjour -> 继续 refine`

---

## Current Gaps

1. Compare 还不是决策工具
   - 当前 compare 只展示基础分数和摘要，缺少 source、freshness、Bonjour link、top evidence、推荐结论。
   - compare pool 的命令语义不够顺滑，提示和真实支持的动作不完全一致。

2. 缺少导出能力
   - 交互式 CLI 没有 export/save/share 动作。
   - 脚本式 `search --json` 输出过瘦，不足以支撑 shortlist 交付或自动化。

3. Refine / rerank 还不够自然
   - 当前主要是自由文本改写条件，缺少 must-have / exclude / prefer-fresh / “像 2 号但更技术” 这类表达能力。
   - rerank 维度只有 overall / tech / project / location，不支持 freshness/source/evidence strength。

4. Query-aware 判断能力还不够强
   - why、compare、detail 更多是在说“通用画像”，不是“为什么适合这次搜索”。
   - 生成的 summary/highlights 仍可能偏英文，不够贴合中文 CLI 体验。

## Success Criteria

1. 用户输入宽查询，如 `杭州`，仍能稳定得到可比较的 shortlist。
2. 用户执行 compare 时，系统不仅展示候选人信息，还能帮助回答“谁更值得点开，为什么”。
3. 用户无需离开 CLI，就能把当前 shortlist 导出为 Markdown / CSV / JSON。
4. 用户可以用自然语言继续 refine，如：
   - `去掉销售`
   - `更看重最近活跃`
   - `像 2 号但更偏后端`
5. rerank-only 指令优先重排，不必每次都重新检索。
6. detail / why / compare 的解释对“当前查询”保持敏感，而不是只输出通用画像。
7. `search --json` 可输出足够完整的数据，支撑后续 automation / export / share。
