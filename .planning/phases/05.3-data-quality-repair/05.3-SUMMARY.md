## Objective

Fix the pipeline/derivation problems causing sparse data quality in CLI search results.

**Root Cause Analysis:**
- **headline (3.2%)**: Merger only lifted `profile.headline`, ignored `bio/summary` (90% filled)
- **facetSource (3.2%)**: Only extracted from evidence; payload stringified broke extraction
- **facetRole (3.2%)**: Extractor only read `primaryHeadline` + `job_signal`, ignored `summary/bio`

**Key Insight:** Not a data source problem. It was a pipeline/derivation problem.

## P0: Evidence Pipeline Repair

### Problem
- `source_profiles.raw_payload` / `normalized_payload` stored as JSON string root instead of object
- Evidence extraction from profiles failed silently
- `facetSource` only derived from sparse evidence

### Solution
1. Added `coerceJsonObject` helper to parse stringified JSON
2. Applied coercion in all read/write paths
3. Added identity/source-profile fallback for `facetSource`
4. Created repair worker for historical data

### Files Modified
- `packages/db/src/repositories.ts` - coercion helper + upsert fix
- `packages/workers/src/evidence-storage.ts` - coerce before extraction
- `packages/identity/src/merger.ts` - coerce in getNormalizedProfile
- `packages/search/src/index-builder.ts` - identity fallback for facetSource
- `packages/workers/src/search-index-worker.ts` - pass source hints
- `packages/workers/src/source-profile-repair.ts` - NEW: repair worker
- `apps/worker/src/cli.ts` - `repair-source-payloads` command

## P1: Headline Backfill

### Problem
- Merger only lifted `profile.headline` (3.3% filled)
- `bio` (90%) and `summary` (100%) ignored

### Solution
