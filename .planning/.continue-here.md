# Continue Here — Launcher Command Fix + Visual Simplification Brief

## Current Batch

User feedback after Phase 8:

- Visual style is still not unified.
- Mascot is not visible enough.
- Slash commands need default discoverability/completion behavior.
- Launcher advertised `/help`, but typing `/help` was treated as invalid input.
- Launcher has too much noisy metadata (`legacy`, `local cache`, timestamps, repeated state).

## Work Done In This Batch

- Fixed launcher command parsing so `/help` and `?` are real launcher actions.
- Made advertised launcher commands consistent:
  - `/task` and `/workboard` show selected task workboard.
  - `/transcript` shows selected task transcript.
  - `/tasks` refreshes the task list.
  - `/memory`, `/new`, `/resume`, and quit aliases remain supported.
- Updated launcher invalid-input copy to point users to `/help`.
- Added launcher parser regression coverage for `/help`, `?`, `/workboard`, `/transcript`, and `/tasks`.
- Created Claude Code visual handoff brief:
  - `docs/superpowers/specs/2026-04-24-cli-visual-simplification-claude-brief.md`

## Design Direction For Claude Code

Claude Code should take the next visual pass using `frontend-skill`.

The preferred direction is a calm operator-console style:

- one visual anchor per screen;
- compact launcher rows;
- mascot as a visible single-line navigator (`Seeku` + terminal-safe glyph), not a new panel;
- slash command help through `/help` and compact suggestions instead of a long permanent footer;
- legacy/cache/timestamp/debug metadata hidden by default and moved into `/task`, `/workboard`, `/help`, or debug/detail views.
