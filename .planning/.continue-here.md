## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Blanket historical frontier exclusion | Pulling all old frontier/import artifacts into default excludes destroys the experiment signal and can zero the seed file before probe. | blocking | Restrict auto excludes to prior fresh-auth-consumed artifacts only, then verify attribution in the blocked-handle report before further tuning. |
| Misattributing zero-output runs | The first attribution pass looked only at recent seed dedupe, but the verified zero-output case came from historical auto excludes. | blocking | Always inspect both `blocked-reasons.json` and `recent-seed-blocked-handles.json` after a zero-output run. |
| Trusting placeholder planning docs | Some legacy `.planning` files still contain `TBD`/placeholder markers and can look more complete than they are. | advisory | Treat placeholder-bearing planning docs as noise unless they were refreshed in the current workstream. |
