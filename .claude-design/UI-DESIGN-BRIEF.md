# Seeku UI设计需求

## 产品概述

**Seeku** 是一个中文AI人才搜索引擎，帮助招聘方和技术创始人通过项目证据（代码、开源贡献、项目经历）找到真正匹配的候选人，而非依赖简历关键词匹配。

**核心价值主张：** 通过真实证据找人，而非简历包装。

## 目标用户

1. **招聘方/HR** - 需要高效筛选AI人才，重视专业性和可信度
2. **技术创始人/工程师** - 懂技术，重视代码质量和实际能力
3. **中文用户为主** - 需要优化中文显示和阅读体验

## 功能模块

### 1. 搜索首页
- 自然语言搜索框（支持中文输入）
- 示例搜索词提示
- 简洁但有视觉冲击力

### 2. 搜索结果页
- 候选人卡片列表
- 每张卡片包含：姓名、职位、公司、匹配分数、匹配原因标签、证据预览
- 点击卡片弹出详情模态框

### 3. 候选人详情弹窗
- Evidence Tabs：项目、代码仓库、社交链接、职业信号
- 结构化展示证据内容

### 4. 管理后台
- 同步状态、评估指标、手动触发按钮
- 简洁的数据展示

## 设计要求

### 核心需求

**结合两套方案的优点：**

1. **混合主题** - 采用Electric Studio的深浅搭配，但要有更丰富的层次
   - 深色头部导航 + 浅色主体内容
   - 可以考虑渐变过渡或区块划分增加层次感

2. **双栏布局** - 采用Professional Navy的功能布局
   - 左侧：筛选器面板（可折叠）
   - 右侧：搜索框 + 结果列表
   - 移动端：筛选器折叠为底部抽屉

3. **中文优化**
   - 字体：使用适合中文的字体组合（Noto Sans SC、思源黑体等）
   - 排版：考虑中文的行高和字重
   - 标签和提示文案使用中文

4. **科技感 + 专业感平衡**
   - 不是纯商务风（太传统）
   - 不是纯科技风（太炫）
   - 要有AI产品的前沿感，但保持可信度

### 视觉元素

1. **配色**
   - 主色：深蓝或海军蓝（专业、可信）
   - 强调色：青绿或电光蓝（科技感、活力）
   - 中性色：暖灰或冷灰（平衡、舒适）
   - 避免：纯黑、纯白、高饱和度色彩

2. **字体**
   - 中文标题：思源黑体/Noto Sans SC 600-900
   - 中文正文：思源黑体/Noto Sans SC 400-500
   - 英文/数字：可以用不同字重区分

3. **卡片设计**
   - 不要太扁平，要有微妙的层次（阴影、边框、背景色变化）
   - 悬浮状态要有明显反馈
   - 匹配分数要突出显示
   - 标签使用强调色

4. **图标**
   - 使用线性图标（Lucide风格）
   - 与整体风格统一

### 交互要求

1. **搜索框**
   - 输入时自动聚焦
   - 有loading状态
   - 支持清空

2. **筛选器**
   - 实时筛选（无需点击确认）
   - 已选筛条件显示为标签，可快速移除
   - 支持折叠/展开

3. **候选人卡片**
   - Hover效果：上浮、阴影增强
   - 点击：弹出详情模态框
   - 证据预览：最多显示3项

4. **详情模态框**
   - 居中显示
   - 点击背景或ESC关闭
   - Tab切换证据类型
   - 响应式适配

### 响应式

- 桌面端（>1024px）：双栏布局
- 平板端（768-1024px）：筛选器折叠，单栏
- 移动端（<768px）：全宽卡片，筛选器底部抽屉
